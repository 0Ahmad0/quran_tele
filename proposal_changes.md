# مقترح التعديلات - بوت ورد القرآن

## 1. ✅ إظهار عدد الختمات المقروءة

### الوضع الحالي:
- يوجد `completed_khatmas` و `khatma_read_count` في قاعدة البيانات
- عند نهاية الختمة، تظهر أزرار "قرأت الختمة" / "لم أقرأ الختمة"
- `completed_khatmas` يزيد عند الضغط على "قرأت" فقط
- عند /status لا يظهر عدد الختمات المقروءة

### المطلوب:
- إظهار `عدد الختمات المقروءة` في `/status` و `/admin_stats`
- عدد الختمات = `completed_khatmas` (الختمات التي تم تسليمها)
- إظهارها بالشكل: `✅ عدد الختمات المقروءة: X`

## 2. 🚫 إخفاء "وضع النص فقط" من الكابشن

### الوضع الحالي:
- سطر `📝 وضع النص فقط` يضاف إلى caption الإرسال عند اختيار "نص فقط"
- يظهر في كل مرة يُرسل فيها الورد

### المطلوب:
- **إخفاء "📝 وضع النص فقط" من caption الورد المرسل**
- إبقاء الخيار متاحاً في الإعدادات فقط (نوع الإرسال)
- المستخدم العادي لا يرى ملاحظة "نص فقط" في الرسالة اليومية

### طريقة التنفيذ:
- إزالة `caption += get_text(language, "text_only_note")` من `send_daily_quran`
- بقاء الإعداد في /send_type كما هو
- عند /status يظهر نوع الإرسال الحالي طبيعي

## 3. 🤲 أدعية ديناميكية (قابلة للتعديل من الأدمن)

### الوضع الحالي:
- الأدعية مخزنة بشكل ثابت (hardcoded) في `utils.py` كقائمة `DUAS`
- لا يمكن إضافتها أو تعديلها إلا بتغيير الكود

### المطلوب:
- تخزين الأدعية في ملف JSON منفصل (`duas.json`)
- إضافة أمر للأدمن فقط لإضافة دعاء جديد:
  - `/add_dua <نص الدعاء>`
- يمكن رفع ملف `duas.json` مباشرة من الأدمن
- عند إرسال دعاء عشوائي للجميع، يتم السحب من هذا الملف

### هيكل الملف المقترح:
```json
[
    "اللهم اجعل القرآن ربيع قلوبنا...",
    "اللهم علمنا من القرآن ما جهلنا..."
]
```

**ملاحظة:** الأدعية الحالية تبقى كما هي، فقط تُنقل من utils.py إلى ملف JSON.

## 4. 💾 حل قاعدة البيانات - تحديث بدون توقف الخدمة

### المشكلة الحالية:
- SQLite على منصات الاستضافة المجانية (مثل Render) يُحذف عند إعادة التشغيل
- عند رفع نسخة جديدة من الداتابيز، يتوقف البوت مؤقتاً ويضطر المستخدمون لإعادة الإعداد

### الحل المقترح (Turso + JSON Export/Import):

#### أ. Turso (قاعدة سحابية):
- استخدام `libsql_client` للاتصال بـ Turso عند توفر متغيرات البيئة
- `DATABASE_URL` و `DATABASE_AUTH_TOKEN` لربط Turso
- البوت يعمل مع Turso مباشرة دون تغيير في سلوك المستخدمين
- إذا لم تتوفر متغيرات Turso، يستخدم SQLite المحلي تلقائياً

#### ب. JSON Export/Import (نسخ احتياطي يدوي):
- `/export_json` - تصدير جميع بيانات المستخدمين إلى ملف JSON
- `/import_json` - استقبال ملف JSON ودمجه مع قاعدة البيانات الحالية (upsert)
- لا حذف للبيانات الحالية - فقط تحديث
- البوت لا يتوقف عن العمل أثناء التحديث

## 5. 📝 منطق تسجيل الختمة (لم تقرأ الختمة)

### الوضع الحالي:
- عند نهاية الختمة، يظهر للكل: "هل قرأت الختمة كاملة؟" مع زرين ✅/❌
- إذا ضغط "لم أقرأ"، تظهر رسالة للجميع "📝 تم تسجيل أنك لم تقرأ الختمة"
- **المشكلة:** الرسالة عامة للجميع (في حالة المجموعات، الكل يرى أن فلان لم يقرأ)

### آلية العمل المقترحة:

```
عند انتهاء الختمة:
  ↓
1. إرسال رسالة خاصة لكل مستخدم (وليس للمجموعة):
   "🎉 هنيئاً لك! تم ختم القرآن... هل قرأت الختمة؟"
   مع زرين: ✅ قرأت / ❌ لم أقرأ
  ↓
2. إذا ضغط "قرأت":
   ✓ يزيد completed_khatmas + 1
   ✓ يزيد khatma_read_count + 1
   ✓ رسالة خاصة: "تم تسجيل ختمتك ✅"
  ↓
3. إذا ضغط "لم أقرأ":
   ✗ لا يزيد شيء
   ✗ رسالة خاصة: "📝 لم تقرأ الختمة"
   ✗ سجل في قاعدة البيانات كـ "لم يقرأ هذه الختمة"
   ✗ (اختياري) إرسال تنبيه للأدمن فقط أن X لم يقرأ
  ↓
4. كلا الحالتين:
   ✓ يزيد khatma_number + 1
   ✓ يرجع page = 1
```

### حل إضافي مقترح:
- إضافة حقل `khatma_unread` في قاعدة البيانات لتتبع عدد الختمات غير المقروءة
- إضافة أمر للأدمن: `/unread_stats` - يعرض عدد الذين لم يقرأوا آخر ختمة
